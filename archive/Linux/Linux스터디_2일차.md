### 유닉스부터 리눅스까지

![리눅스의역사](https://github.com/user-attachments/assets/721b3659-01f6-4192-a29b-d0e2ad1db04d)

리눅스는 유닉스라는 조상으로부터 내려와<br/>
탄생하게 되었는데, 유닉스나 리눅스는 둘 다 운영체제이다.<br/>
보통 많은 사람들은 대부분 Windows를 쓰는데,<br/>
여기서 의문점은, '왜 리눅스를 배우냐?' 이다.<br/>
<br/>
![서버사진](https://github.com/user-attachments/assets/4e250b65-1f71-478a-bf40-6e90adc0da6a)

요즘 운영체제는 '다중사용자'와 '멀티태스킹'이 가능하다.<br/>
서버를 다루는 개발자면 리눅스는 거의 필수로 알아야하는데,<br/>

- 다중사용자 : 컴퓨터를 사용하는 여러명의 사용자
- 멀티태스킹 : 컴퓨터가 한번에 여러 작업을 처리하는 것.

<br/>
<br/>

요즘은 이 두가지 작업이 가능하기 때문에,<br/>
서버를 다루고, 중요한 정보를 다루는 회사나 개발자들은<br/>
더욱 많은 보안성을 요구한다.<br/>
<br/>
리눅스는 윈도우와 다르게 운영체제의 소스코드를 제공한다.<br/>
윈도우를 배포하는 마이크로소프트는 소스코드를 절대 공개하지 않지만,<br/>
이러한 단점으로 프로그램의 결함이 있을 때,<br/>
신속하게 처리하기 어렵다.<br/>
<br/>
리눅스는 자유롭게 소스코드를 제공하고 , 이러한 소스코드를 이용해서<br/>
다양한 it기업에서 여러 배포판을 내놓는 운영체제이다.<br/>
<br/>
그래서 프로그램에 결함이 있을 때, 비교적 빠르게 결함을 수정하여<br/>
새로운 프로그램이 배포된다.<br/>
<br/>
보안이 중요한 기업들은, 리눅스의 이러한 이점들 덕분에<br/>
빠르게 취약점이 수정되는 리눅스 체제를 많이 사용한다.<br/>

<br/>
<br/>
<br/>

## 운영체제

우리가 대부분 컴퓨터를 쓸 때, 윈도우 라는 운영체제를 쓴다.<br/>
많은 사람들이 윈도우 , 운영체제 라는 단어는 알지만,<br/>
정확히 이것이 무얼 하는지 역할을 모른다.<br/>
<br/>
대충 CPU , GPU가 좋으면 게임이 잘 돌아가고,<br/>
이러이러한 기능은 운영체제 때문에 호환이 안되고<br/>
<br/>
이렇게 겉핥기로만 알고 있다.<br/>
물론 필자도 마찬가지다.<br/>
<br/>
먼저 운영체제는<br/>

- 프로세스 및 메모리 관리
- 파일 시스템 관리
- 입출력 관리
- 자원 관리 및 보호
- 사용자 인터페이스(GUI) 제공

등의 기능을 한다.<br/>
<br/>
<br/>
이번 글에서는 이중 주요 기능만 다뤄볼 것이다.<br/>
더 깊숙히 들어가면 필자의 머리가 감당하지 못한다.
<br/>
운영체제가 다루는 기능을 그림으로 나타내면 이렇다.<br/>

![운영체제의 기능](https://github.com/user-attachments/assets/0813bda4-d791-4e04-b4b4-1a47b0abda65)

<br/>
<br/>

### 1 . 프로세스 및 메모리 관리

이번에 상술할 내용은,<br/>
운영체제 속 커널이 수행한다.<br/>
<br/>
정확히는,<br/>

- 모놀리식 커널 : 운영체제의 모든 기능을 포함한 커널
- 마이크로 커널 : 운영체제의 핵심기능(프로세스 관리,메모리 관리 등)만 제공

이러한 커널 종류가 있다.<br/>
운영체제마다 커널의 종류도 다른데,<br/>

- 모놀리식 커널 : 리눅스 , 유닉스 , 윈도우 11
- 마이크로 커널 : Mac OS X

커널의 종류는 이렇게 사용한다.<br/>
우리가 배울 것은 리눅스이기 때문에,<br/>
모놀리식 커널을 기반한 운영체제이다.<br/>
<br/>
<br/>
첫 번째로 '프로세스 관리'를 알아볼 것이다.<br/>
많은 사람들이 프로그램이 무엇인지는 많이 알 것이다.<br/>
<br/>
정의를 하자면,<br/>
<br/>
컴퓨터에서 특정 작업을 '수행하는' 일련의 명령어의 모음이다.<br/>
하지만, 이러한 프로그램은 실행되지 않으면 계속 프로그램인 것이다.<br/>
<br/>

![프로그램프로세스](https://github.com/user-attachments/assets/0bc0c73b-7d63-476e-b850-6df367dcd4a4)

프로그램이 다음에 서술할 '메모리'에 올라가서<br/>
동작을 하게 되면, 그 놈은 '프로세스'라는 것으로 바뀐다.<br/>
이 과정에 게임에서 흔히 나오는 '로딩' 이라는 개념이다.<br/>
<br/>
이러한 프로세스들을 관리 하는 것도 운영체제의 역할이다.<br/>

![프로세스 관리](https://github.com/user-attachments/assets/2c4394a4-bfa9-4a64-b415-b0f129466a41)

이렇게 하드디스크에 있던 프로그램들을 메모리에 옮길 때,<br/>
각 프로세스들이 메모리에 있을 수 있는 공간을 관리하고,<br/>
프로세스를 생성 , 실행 , 일시중단 , 재개 등등을 담당한다.<br/>
<br/>
메모리에 올라온 프로세스 중 CPU가 프로세스를 실행시키고,<br/>
실행하지 않는 프로세스는 대기 시킨다.
<br/>

![ram](https://github.com/user-attachments/assets/9f96fb4b-db16-48da-a3f3-60f069fdf383)

RAM 예시 자료)

- RAM : Random Access Memory
<br/>사용자가 자유롭게 내용을 읽고 쓰고 지울 수 있는 기억장치

<br/>
<br/>

메인 메모리는 RAM 이라고 하는데,<br/>
고사양 게임을 하는 유저들이 고성능 RAM을 사용해야하는 이유가<br/>
RAM의 용량이 많으면 프로세스를 위해서 더욱 많은 공간을 사용할 수 있다.<br/>
<br/>
그러므로 고사양 RAM을 사용하면 위에 서술한 프로세스 관리가<br/>
더욱 원활하게 가능하다.
<br/>
<br/>
<br/>
<br/>

### 2. 파일 시스템 관리
'파일'이란 정보를 담기위한 논리적인 저장 단위이다.<br/>
레코드나 블록 단위로, 비휘발성 보조기억장치에 저장된다.<br/>
위에 서술했던 'RAM'은 읽고 쓰기가 가능하지만<br/>
컴퓨터의 전원이 꺼지면 안에 저장되었던 데이터가 전부 지워진다.<br/>

![컴퓨터전원꺼버리기](https://github.com/user-attachments/assets/17e2b61b-b709-42e0-9bbe-929e1fcbe40c)

예전 뉴스에서 방영되었던<br/>
PC방 컴퓨터의 전원을 꺼버리는 방송이 있었는데,<br/>
게임같이 현재 실행중인 프로그램들은 RAM에<br/>
한시적으로 저장이 되기 때문에<br/>
전원을 꺼버려 게임 진행상황이 모두 날아가<br/>
당시 출연한 사람들의 빡침이 화면밖으로 느껴지기도 했다.<br/>
<br/>
<br/>
<br/>

#### 비휘발성 보조기억장치 

<br/>
그래서 운영체제나 지워지면 안되는 중요한 프로그램은,<br/>
ROM에 저장된다.
<br/>
<br/>

![rom](https://github.com/user-attachments/assets/b78bc456-5554-4761-8819-379d9daa50c3)

- ROM : Read Only Memory <br/>
오직 읽기만 가능한 보조기억장치

<br/>
이 장치는 RAM과 다르게 비휘발성 보조기억장치이다.<br/>
<br/>

- 비휘발성 보조기억장치 : 전원이 꺼져도 저장된 데이터가 지워지지 않음.
- 휘발성 보조기억장치 : 전원이 꺼지면 안에 저장된 데이터가 지워짐.

<br/>
<br/>
<br/>

#### 파일 시스템

운영체제 , 파일 메타 데이터등의 저장을 맡고,<br/>
프로그램 접근 및 저장을 위한 기법을 제공한다.<br/>
<br/>

- Linux : xfs, ext4, ext3, jfs, ReiserFS 등등

- Windows : FAT, FAT32, NTFS 등등

- UNIX : UFS(Unix File System), ZFS

- IOS : APFS(Apple File System), HFS+

<br/>
<br/>

각 운영체제마다 파일 시스템의 종류는 이러하다.<br/>
예를 들어, Windows의 FAT32 라는 파일 시스템은<br/>
옛날 윈도우에서 사용했던 파일 시스템인데,<br/>
여기서 32는 32bit용이다.<br/>
<br/>
요즘 컴퓨터는 대부분 64bit 체제라 호환이 잘 안된다.<br/>
결정적인 단점은, 4GB가 넘는 파일을 읽지 못한다.<br/>
<br/>
요즘은 4GB넘는 파일이 흔하지만, 옛날에는 이 정도의 파일 크기가<br/>
흔하지 않다보니, 파일 시스템도 사양이 많이 낮았다.<br/>
<br/>
현재는 윈도우의 파일 시스템으로 'NTFS'를 많이 사용한다.<br/>

![NTFS시스템](https://github.com/user-attachments/assets/18d42fbb-19a0-40e3-ad1c-5b065c30db5a)

필자는 윈도우 11을 쓰고 있는데, C 드라이브의 파일 시스템이<br/>
NTFS인걸 볼 수 있다.<br/>
<br/>
각 운영체제마다 각기 다른 파일 시스템을 구축하고 있는데,<br/>
다른 운영체제끼리 파일을 주고 받았을 때, 파일을 열지 못하는 경우가 있다.<br/>
<br/>
<br/>
<br/>
<br/>


### 3. 사용자 인터페이스(GUI) 제공
이 단원을 학습하기 전에 먼저,<br/>
Interface(인터페이스) 에 관해 배울 필요가 있다.<br/>
<br/>
Inter(상호간의) + face(면 , 얼굴)<br/>
상호간의 대면한다는 뜻인데,<br/>
사용자와 컴퓨터를 이어준다는 뜻이다.<br/>
<br/>
대표적인 인터페이스는,<br/>
GUI , CUI 가 있는데,

- CUI : Character User Interface
- GUI : Graphical User Interface<br/>
<br/>
가 있다.<br/>

CUI는 오직 글자만 있는창이고,<br/>
CLI(Command - Line - Interface)라고도 한다.

<br/>

![cui 화면](https://github.com/user-attachments/assets/6caec605-7c24-41db-86ac-0044e2c28813)

운영체제가 등장하기 전에는,<br/>
사용자가 일일히 명령어를 입력하여<br/>
컴퓨터와 상호작용 하였다.<br/>
<br/>
겁나게 복잡하고 어렵다보니, 일반인은<br/>
컴퓨터에 쉽게 접근하지 못했다.<br/>
이러한 컴퓨터의 세계에서 GUI의 등장은 혁명과도 같았다.<br/>
<br/>
<br/>

#### GUI (Graphical User Interface)

한국어로 '그래픽 사용자 인터페이스' 이다.<br/>
그래픽카드 할때 그 그래픽 맞다.<br/>
<br/>

![gui 화면](https://github.com/user-attachments/assets/d1c4c2db-1e93-4004-96d2-e48e2dc50a3a)

위와같이 글자만으로 컴퓨터와 상호작용하는<br/>
어려운 방식이 아니라, 그림과 버튼 등<br/>
보자마자 클릭만 하면 상호작용이 이루어지는<br/>
쉬운 방식이다.<br/>
<br/>
거의 모든 운영체제가 GUI 방식을 채택하여<br/>
사용자에게 편의를 주고있다.

<br/>
<br/>
<br/>
