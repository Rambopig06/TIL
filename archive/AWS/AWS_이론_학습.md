# AWS의 시작

AWS : Amazon Web Service<br/>

AWS 풀 네임의 의미처럼 아마존에서 제공하는<br/>
클라우드 컴퓨팅 프로그램이다.<br/>
<br/>
AWS 에 관해 알아보기 전, 클라우드 컴퓨팅에 관해 알아보도록 하자.

## 클라우드 컴퓨팅

클라우드 컴퓨팅은, 사용자가 굳이 어려운 통신장비 , 네트워크를<br/>
복잡하게 일일이 관리할 필요 없이 인터넷만 있으면<br/>
파일 등을 불러와 더 쉽게 작업할 수 있는 혁신적인 방식이다.<br/>
<br/>
클라우드는 구름이라는 뜻인데,<br/>
이 클라우드의 유래는 , 사용자가 굳이 어렵게 구름 속 내부를 알 필요 없이<br/>
구름 안에 있는 자원을 사용할 수 있다는 의미에서 나왔다.<br/>
<br/>
![크롬북](https://github.com/user-attachments/assets/64718857-98ca-478c-9199-c6a903d81104)

대표적으로 클라우드 컴퓨팅이 쓰인 제품인 크롬북을 알아보겠다.<br/>

![크롬북클라우드](https://github.com/user-attachments/assets/33078181-cf6b-4e9e-b2be-febb31729e9d)

이제 이것도 꽤 옛날이긴 하지만<br/>
뉴스에서 나올 정도로 잠깐 어그로가 끌렸던 제품이다.<br/>
크롬북은 전원을 켰을 때 운영체제가 돌아가는 대신<br/>
크롬 브라우저가 실행된다.<br/>
<br/>
작고 얇은 노트북에 저장공간을 만드는 대신<br/>
아싸리 인터넷에 연결해서 서버에서 왠만한걸 다 다운받아 쓴다는 마인드.<br/>
<br/>
하지만 이런 방식의 단점은 인터넷이 없으면 이 컴퓨터는 그냥 바보가 된다는 것.
<br/>
<br/>

규모가 그리 크지 않은 기업은 클라우드 서버를 사용한다.<br/>
오늘 서술할 AWS 도 클라우드로 서버를 제공하는데,<br/>
<br/>
<br/>

이와 반대되는 개념으로 '온 프레미스' 가 있다.<br/>
기업이 직접 서버를 구축하여 관리하는 방식.<br/>
<br/>
기업의 필요에 따라 입맛대로 서버의 설정을 바꾸는 등<br/>
운영의 자유도가 높지만,<br/>
초기 구축비용과 관리할 인력을 필요로 하는 등 비용이 상당하다.<br/>

신차 장기렌트는 초기에 차량 구매비용을 사용자가 지출하는 것은 같지만,<br/>
자동차의 유지관리는 렌트업체에서 대신 해주기 때문에<br/>
사용자가 차를 직접 관리하는 것 보다 편리하다.<br/>
클라우드 서버도 이러한 이점 때문에 사용한다.

## EC2

![이씨투](https://github.com/user-attachments/assets/761b48bf-6071-4c57-a414-de9196707e98)


EC2는 아마존에서 만든 클라우드 컴퓨팅 서비스 기술이다.<br/>
<br/>

뭔가 엄청 복잡하고 방대한 뜻이 담겨져 있는 것처럼 보이지만,<br/>
그냥 아마존에서 제공하는 클라우드 컴퓨팅 서비스의 사업명이다.<br/>
<br/>
사용자들은 EC2 를 통해 필요한 CPU , 메모리 , 스토리지를 사용할 수 있다.<br/>
웹 서버 , 데이터베이스 , 애플리케이션 서비스 등을 사용할 수 있다.<br/>
다음 파트에서 서술할 인스턴스의 크기를 마음껏 조정하고,<br/>
운영체제와 소프트웨어도 자유롭게 설정할 수 있어 유연성이 높다.<br/>
<br/>
원격 접속시 SSH(리눅스 인스턴스) , RDP(Windows 인스턴스)를 사용할 수 있고,<br/>
여러 위치에 인스턴스를 배포하여 사용할 수 있어 가용성이 높다.<br/>
<br/>
<br/>


## 인스턴스

EC2 를 설명할 때 나왔던 인스턴스란 , <br/>
클라우스 서버를 이용할 때 꼭 알고 사용하는 개념이다.<br/>
<br/>
쉽게 말하면 인스턴스는 컴퓨터이다.<br/>
즉 EC2 는 컴퓨터의 성능 , 크기 , 용량 등을 설정할 수 있는 것이다.<br/>
<br/>
<br/>

## 스케일 업 그리고 스케일 아웃

이제 인스턴스를 설정하며 나오는 용어인,<br/>
스케일 업 , 스케일 아웃을 배워볼 것이다.<br/>
<br/>
### 스케일 업

스케일 업은 인스턴스의 성능을 늘리는 것이다.<br/>
예를 들어 기존에 사용하던 서버 컴퓨터의 메모리가 16GB , 하드가 1TB 일때,<br/>

`(기존 메모리 16GB , 하드 1TB) -> (스케일 업 메모리 32GB , 하드 10TB)`<br/>

이런 방식으로 인스턴스의 사양을 높이는 것이다.<br/>
건물로 따지면 손님을 더 수용하기위해<br/>
건물의 숫자 대신 높이를 더 키우는 것이다.<br/>
이러한 서버 확장 방식을 '수직적 확장' 이라고 한다.<br/>
<br/>
이러한 스케일 업 방식은 장, 단점이 있다.<br/>


```
장점 : 한대의 서버에서 작업하므로 데이터의 정합성이 높음.

단점 : 한대의 서버에서 작업이 이루어짐으로써
만약 서버에 장애가 생기면 복구 전까지 모든 작업이 올 스톱 된다.

사용 예 : 데이터 변동이 빈번하게 일어나는 작업에 사용된다.
```

스케일 업의 반대 개념으로 스케일 다운이 있다.<br/>
말 그대로 반대개념이므로 인스턴스의 사양을 낮추는 것이다.

### 스케일 아웃

인스턴스의 사양을 높이는 방식인 스케일 업과 달리<br/>
스케일 아웃은 인스턴스의 갯수를 늘리는 방식이다.<br/>
<br/>

아까와 똑같이 예를 들자면,<br/>

`(메모리 16GB , 하드 1TB) -> (스케일 아웃후 메모리 16GB , 하드 1TB의 인스턴스 10개)`

이처럼 사양은 그대로지만 동일한 사양의 인스턴스의 갯수를 늘리는 것이다.<br/>
<br/>

가게의 손님이 많아졌을 때 옆 건물까지 사들이고 분점을 만드는 방식이라고 이해하면 된다.<br/>
(필자 집 앞 치킨 가게가 그런 방식을 써서 굉장히 멋있었던 기억이..)<br/>
이러한 방식은 '수평적 확장'이다.<br/>
<br/>
아무튼 이 스케일 아웃도 장,단점이 있다.

```
장점 : 단일 서버에 장애가 생겨 작업이 쌓여있더라도
다른 서버를 이용해 해결하므로 병목 현상이 줄어든다.

단점 : 여러대의 서버에서 한 작업을 처리할 때
동일하지 않은 데이터가 존재하여 작업 시 어려움이 있다.

사용 예 : 데이터 정합성이 떨어지므로 데이터 변동이 많지 않은 작업에서 사용한다.
```

스케일 아웃의 반대 개념으로 스케일 인이 있다.<br/>
필요시 인스턴스의 갯수를 줄여 유연하게 사용하는 것.<br/>
동시에 인스턴스 사용 비용도 절감한다.<br/>
<br/>
이러한 스케일 아웃, 스케일 인 방식은 학생들 수강신청 당일에 많이 사용한다.<br/>


## 로드밸런싱

![로드밸런싱](https://github.com/user-attachments/assets/cbe7e503-2c70-4e85-bad8-e3ffcab2a4d2)

과거에는 한대의 서버로 왠만한 작업은 전부 처리했지만,<br/>
요즘은 사용자가 요구하는 서비스의 용량과 데이터의 다양성과 복잡성이 증가하여<br/>
서버를 증가시켜 이용해야하는 실정이다.<br/>
<br/>
이제 서버를 분산처리하여 이용하는 시대이므로<br/>
분산처리형 서버를 관리할 때, 로드밸런싱은 꼭 필요하다.<br/>
<br/>
로드 밸런싱이란, 특정 서버에 부하가 가지 않게<br/>
서버에 트래픽을 골고루 분배하여<br/>
과부하로 인한 서버 장애가 일어나지 않도록 하는 행위이다.<br/>
<br/>
<br/>
로드 밸런싱에는 대표적인 두 가지 예시가 있는데,<br/>
RR , WRR 이 있다.<br/>
<br/>
<br/>

### 라운드 로빈(Round Robin)

사용자의 요청을 추가적인 연산없이<br/>
알고리즘에 서버들에 순차적으로 요청을 분산하는 방식이다.
<br/>
주로 모든 서버의 사양이 전부 동일할 때 사용하는 방식.<br/>
<br/>
<br/>

### 가중 라운드 로빈(Weighted Round Robin)

이름처럼 사용자의 요청량과 서버의 사양에 비례하도록<br/>
로드 밸런싱을 하여 분배하는 방식이다.<br/>
<br/>
A 사용자의 요청량은 3 , B 사용자의 요청량은 1 이라고 했을 때,<br/>
<br/>
당연히 A 사용자의 작업이 더욱 높은 사양의 인스턴스를 요구하므로<br/>
A 사용자는 성능 및 용량이 더 뛰어난 서버에<br/>
요청을 전달하는게 효율적이다.<br/>
<br/>
이럴 때 가중 라운드 로빈 기술이 사용된다.<br/>
<br/>
특정 서버의 사양이 높거나 낮을 때 사용하는 방식.<br/>
<br/>
